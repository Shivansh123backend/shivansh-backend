SHIVANSH — Add New Features to Existing Dashboard (Addendum)

Backend: https://shivanshbackend.replit.app/api — all requests need Authorization: Bearer <token>

Add the following to the existing Shivansh dashboard UI:

1. CALL LOGS & CALLS — Add Download, Transcript, Summary, Disposition
In the Call Logs table (GET /api/call-logs), add a new column Actions with two download buttons per row:

TXT button → opens GET /api/call-logs/:id/export?format=txt with auth header → triggers browser file download named call-log-{id}.txt
PDF button → opens GET /api/call-logs/:id/export?format=pdf with auth header → triggers browser file download named call-log-{id}.pdf
To download with auth (since the header can't be sent via <a href>), use this pattern:

const res = await fetch(url, { headers: { Authorization: `Bearer ${token}` } });
const blob = await res.blob();
const a = document.createElement('a');
a.href = URL.createObjectURL(blob);
a.download = filename;
a.click();

Add a Direction column to both Call Logs and Calls tables — badge: inbound (cyan) | outbound (purple).

Disposition column — show as a colored badge:

interested → green
not_interested → red
vm → amber
no_answer → grey
busy → orange
connected → blue
callback_requested → cyan
transferred → purple
completed → teal
Clickable row / detail drawer — clicking any call log row opens a side drawer with:

All metadata (phone, campaign, direction, status, disposition, duration, timestamp)
Recording section: if recording_url is not null, show an HTML5 <audio controls> player with the recording URL. If null, show "No recording available"
AI Summary section: show summary text in a card. If null, show "Summary not yet generated"
Transcript section: show transcript text in a scrollable code-style block (monospace, dark background). Format as readable conversation if it contains AI Agent: / Caller: lines. If null, show "Transcript not available"
Manually Tag Disposition button → opens a small dropdown/popover with disposition options → on select: PATCH /api/call-logs/:id/disposition body: { "disposition": "interested" } → update row in table immediately
Same drawer for Calls table (GET /api/calls) — same fields, but download uses /api/calls/:id/export?format=txt|pdf and disposition update uses PATCH /api/calls/:id/disposition.

Campaign detail — in the edit form, ensure the Max Concurrent Calls slider/input (1–20) is visible and editable. Save with PATCH /api/campaigns/:id body { "maxConcurrentCalls": N }.
