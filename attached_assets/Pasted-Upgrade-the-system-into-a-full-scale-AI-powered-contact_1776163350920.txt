Upgrade the system into a full-scale AI-powered contact center dialer combining Vicidial (dialer logic) and VAPI (AI calling) capabilities.

---

GOAL:

Build a production-grade dialer with:

* Multi-campaign support
* Parallel + predictive dialing
* Dynamic dialing speed
* Agent + number resource management
* Supervisor controls (listen, whisper, barge)
* Inbound + outbound routing
* Answer detection (AMD)
* Spam/risk protection
* Queue-based stability
* Advanced CDR + real-time monitoring

---

========================

1. CAMPAIGN SYSTEM
   ========================

Add fields:

* dialing_mode (manual, progressive, predictive, preview)
* max_parallel_calls
* dialing_ratio
* dialing_speed (calls per second)
* assigned_agents (array)
* assigned_numbers (array)

---

========================
2. DIALING MODES
================

manual:

* trigger single call

progressive:

* dial next lead when agent free

predictive:

* dial leads ahead of agent availability

preview:

* show lead before dialing

---

========================
3. GLOBAL NUMBER POOL
=====================

Table: numbers

Fields:

* id
* phone_number
* status (available, busy, blocked)
* usage_count
* spam_score
* last_used_at

---

RULES:

* Numbers are NOT permanently tied to campaigns
* Any campaign can use available numbers
* If number in use → cannot be reused
* After call ends → return to pool

---

========================
4. NUMBER AVAILABILITY SYSTEM
=============================

Before dialing:

* check number.status = available

On call start:

* number.status = busy

On call end:

* number.status = available

---

========================
5. SPAM / RISK ENGINE
=====================

Rules:

* usage_count++
* if usage_count > threshold → mark "risk"
* if spam_score high → mark "blocked"

Return UI flags:

* 🟢 available
* 🟡 risk
* 🔴 blocked

Blocked numbers:
→ cannot be used

---

========================
6. REDIS CALL QUEUE (CRITICAL)
==============================

Key:

call_queue:{campaign_id}

---

FLOW:

* push leads into queue
* worker processes queue

---

========================
7. PARALLEL DIALING ENGINE
==========================

Steps:

1. fetch available agents
2. fetch available numbers
3. calculate target_calls

target_calls = agents_available * dialing_ratio

limit:

if target_calls > max_parallel_calls:
→ cap to max_parallel_calls

---

========================
8. DIALING SPEED CONTROL
========================

Dial calls in batches:

dialing_speed = calls per second

Example:

dialing_speed = 5
→ 5 calls every second

Implement:

* batch execution loop
* delay 100–300ms between calls

---

========================
9. ADAPTIVE SPEED ENGINE
========================

If:

* agents idle → increase speed
* voicemail high → reduce speed
* failures increase → slow down

---

========================
10. ANSWER DETECTION (AMD)
==========================

Use Telnyx AMD:

On answer:

if human:
→ AI / agent flow

if voicemail:
→ play voicemail script
→ hangup

Store:

answer_type = human / voicemail
